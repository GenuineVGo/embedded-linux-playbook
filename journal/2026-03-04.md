# Journal — 2026-03-04

## Objective
- Stabiliser le lab réseau (OnePlus WAN) + rendre le workflow Buildroot RPi4 reproductible (defconfig versionné, overlay, SSH, IP fixe).

- Réseau RPi4 validé en IP fixe : `192.168.137.6/24`, GW `192.168.137.1`, DNS `192.168.137.1` via `S40network` (overlay), lien 1Gbps.
- SSH RPi4 OK (Dropbear) : connexion depuis Windows via `ssh rpi4` (alias) + `authorized_keys` en place.
- Mise en place d’un LAN stable via Windows ICS (MS-A2 routeur/NAT) :
  - LAN: `192.168.137.1/24` (interface Ethernet vers le switch, pas de gateway — normal)
  - WAN: OnePlus tether (interface Ethernet 7) avec route par défaut.
- CM3588 : bascule en IP fixe sur le LAN ICS via NetworkManager :
  - IP `192.168.137.23/24`, GW `192.168.137.1`, DNS `192.168.137.1`
  - SSH Windows → CM3588 OK (`ssh cm3588` alias).
- Buildroot RPi4 reproductible :
  - defconfig versionné : `buildroot/external/configs/rpi4_64_defconfig`
  - build out-of-tree persistant : `~/.cache/br-out/rpi4_64`
  - images produites : `sdcard.img` + `boot.vfat` + `rootfs.ext4`
  - overlay injecté (preuve : `target/etc/init.d/S40network` présent)
  - dropbear présent (preuve : `target/usr/sbin/dropbear` présent)
- Flash SD OK via `make flash-buildroot-rpi4` (script demande `YES` en majuscules).
- WSL2 : FT232R (0403:6001) attaché via `usbipd`, accès OK à `/dev/ttyUSB0` (picocom).

## What broke
- Non-reproductibilité Buildroot (au début) :
  - confusion entre `.config` in-tree vs `O=...`
  - `BR2_OUT` volatile en `/dev/shm` (perte après reboot)
  - external tree sans defconfig au départ → build “générique” (`rootfs.tar` seul) au lieu d’une image SD.
- mDNS côté Windows :
  - `cm3588.local` restait résolu sur une ancienne IPv4 (`10.35.205.47`) → SSH timeout sur `.local`.
- Réseau “OnePlus patron” instable :
  - tether Android change de subnet sans prévenir → incompatible avec IP fixes lab.
- `systemctl suspend` sur CM3588 : refus (non pas permissions, mais cible systemd masquée).
- `ping` ICMP bloqué via ICS/tether alors que TCP/DNS OK (`apt update/upgrade` OK).
- Changement de host key après reflash RPi4 : alerte SSH “REMOTE HOST IDENTIFICATION HAS CHANGED” (résolu par purge known_hosts).
- Dropbear initialement inaccessible : mot de passe root inconnu → passage à auth par clé via `/root/.ssh/authorized_keys`.

## Fix / command / reference

### Réseau / ICS
- Windows ICS : LAN forcé en `192.168.137.1/24`, WAN via OnePlus.
- CM3588 IP fixe (NetworkManager) :
  - attention : `nmcli con down/up` peut couper SSH (préférer `nmcli device reapply eth0` ou `nmcli -w 0 ...`).

### Suspend
- Cause : `suspend.target` était **masked**.
  - preuve : `journalctl -u systemd-logind -b` → “Unit suspend.target is masked, refusing operation.”
  - (option) `sudo systemctl unmask suspend.target`

### Buildroot : remise à plat + ancrage defconfig
- Out-of-tree persistant : `O=~/.cache/br-out/rpi4_64`
- Base : `raspberrypi4_64_defconfig`
- SSH : `BR2_PACKAGE_DROPBEAR=y`, `BR2_PACKAGE_TINYSSH=n`
- Overlay : `BR2_ROOTFS_OVERLAY=".../board/rpi4_64/rootfs_overlay"`
- IP fixe RPi4 via overlay : `rootfs_overlay/etc/init.d/S40network`
- Source de vérité : `buildroot/external/configs/rpi4_64_defconfig` (mise à jour via `savedefconfig`)

### Makefile (repo)
- Ajout targets et aide à jour :
  - `print-vars`, `br-menuconfig`, `br-savedefconfig`
  - `buildroot-rpi4` (valeur `BR2_OUT` de `local.mk`)
  - `buildroot-rpi4-fast` (force `/dev/shm`)

### Flash SD
- Script interactif : saisir **YES** (majuscule) sinon abort.
- Après flash : `sudo udisksctl power-off -b /dev/sdX`

### SSH : host key changed
- Windows :
```powershell
ssh-keygen -R 192.168.137.6
ssh rpi4
```

### SSH : auth par clé
- Cible (permissions) :
```sh
chmod 700 /root/.ssh
chmod 600 /root/.ssh/authorized_keys
```
